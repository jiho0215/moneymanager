## §1 Back-reference

Part of epic [compound-learning-elementary](spike-plan.md). See [spike-plan §7](spike-plan.md#7-tickets) for sibling tickets and current status.

## §2 This Ticket's Role in the Big Picture

T1 은 epic의 **walking skeleton**. 모든 후속 티켓 (T2-T6) 이 의존하는 토대를 한 PR에 통째로 만든다. 코드 베이스, schema, RLS 정책, RPC 정의, 인프라 (Vercel preview, Saturday keepalive, Sunday ping bot) 까지 모두 포함. 후속 티켓들은 자기 레이어만 살찌움 — 새 테이블 만들거나 RLS 정책 손대지 않는다.

이 티켓이 머지되면:
- Vercel preview URL이 빈 페이지로 살아 있음 (E2E 테스트 가능)
- DB schema 전체 존재, RLS 정책 적용됨
- `/api/health`, `/api/keepalive` 엔드포인트 응답
- GitHub Actions Sunday ping bot이 cron 실행되어 health endpoint 호출

이게 안 되면 T2 (auth) 의 PIPA consent 흐름도, T3 (domain) 의 마이그레이션 검증도, T4-T6 의 RLS 의존도 못 만든다.

## §3 Relevant API Contract Slice

T1 자체는 사용자용 Server Action 노출 안함. 인프라 endpoint 만:

```typescript
// app/api/health/route.ts (GET)
// 인증: X-Health-Token 헤더 (HEALTH_PING_TOKEN env var)
// 동작: SELECT 1 from families + dummy math problem generation + RLS verify
// 응답: { ok: true, db: 'reachable', mathgen: 'ok', rls: 'enabled' } | 5xx

// app/api/keepalive/route.ts (GET)
// 인증: Vercel Cron의 internal cron secret
// 동작: SELECT 1 FROM families LIMIT 1
// 목적: Supabase free-tier auto-pause 방지

// app/api/cron/monthly-reconcile/route.ts (GET)
// 인증: Vercel Cron internal
// 동작: 모든 active accounts에 대해 reconcile_balance() 호출, drift 시 alarm
```

T2 부터 노출되는 Server Actions (`createFamily`, `getCurrentMathProblem`, `attemptClaim`, `updateAccountSettings`, `chooseCycleEndAction`) 은 이 티켓에서는 **빈 placeholder 함수만** 만들어 둠 (compile pass 위함, 실제 로직 없음).

### Sunday Ping Bot (GitHub Actions)
- `.github/workflows/sunday-ping.yml`:
  ```yaml
  on:
    schedule:
      - cron: '0 10,11,12 * * 0'  # KST 19/20/21 일요일
    workflow_dispatch:
  jobs:
    ping:
      runs-on: ubuntu-latest
      steps:
        - run: |
            response=$(curl -s -o /dev/null -w "%{http_code}" \
              -H "X-Health-Token: ${{ secrets.HEALTH_PING_TOKEN }}" \
              ${{ vars.HEALTH_URL }})
            if [ "$response" != "200" ]; then
              # trigger Resend via repository_dispatch
              curl -X POST ... ${{ secrets.PING_FAILURE_WEBHOOK }}
              exit 1
            fi
  ```
- GitHub의 cron-failures 이메일 알림은 자동 (repository setting)
